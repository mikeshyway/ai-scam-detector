Omkar Carrier Lookup

Omkar API Setup Guide

Configure Omkar Carrier Lookup for live phone-number metadata checks. Store the API key outside source code and fall back to the local processed dataset when live lookup is unavailable.

Setup Steps

  1. Create or sign in to your Omkar Cloud account.
  2. Verify the registered phone number before using live API requests.
  3. Open https://www.omkar.cloud/account/verify-phone and complete verification.
  4. Copy the Carrier Lookup API key from your account.
  5. Add the key to your local .env file or Streamlit secrets.
  6. Restart Streamlit after changing environment variables.
OMKAR_API_KEY=your_omkar_key_here

Open Omkar Carrier Lookup API documentation

Never commit .env, .streamlit/secrets.toml, or real API keys to GitHub.

Lookup Behavior

  1. The app normalizes Malaysian phone numbers into one internal international format.
  2. The app calls Omkar Carrier Lookup only when an Omkar API key is configured.
  3. If Omkar returns an authentication, account-verification, rate-limit, timeout, network, or server error, the app uses the local processed fallback dataset.
  4. If no local record exists, the app returns Unknown, not Safe.

Omkar returns carrier and phone metadata. It does not provide police reports, community scam reports, or a scam probability. Treat carrier metadata as context only and verify sensitive caller claims through official channels.
