Phone Lookup API Setup Guide

Configure PenipuMY, IPQualityScore, or Carrier Lookup for live phone-number checks. Store keys outside source code and fall back to the local processed dataset when a provider is unavailable.

PenipuMY

Setup

  1. Create or sign in to your PenipuMY account.
  2. Generate an API key from account settings.
  3. Add the key to your local .env file.
  4. Restart Streamlit.
PENIPUMY_API_KEY=your_penipumy_key_here

Open PenipuMY API documentation

IPQualityScore

Setup

  1. Create an IPQS account and sign in.
  2. Open Settings and Account Management, then Account Settings.
  3. Copy the API key or private key.
  4. Add it to your local .env file and restart Streamlit.
IPQS_API_KEY=your_ipqs_key_here

Open IPQS Phone Number Validation API documentation

Carrier Lookup

Setup

  1. Create an Omkar Cloud account.
  2. Open the API key page and copy your Carrier Lookup key.
  3. Add it to your local .env file.
  4. Restart Streamlit.
OMKAR_API_KEY=your_omkar_key_here

Open Carrier Lookup API documentation

Recommended .env example

PENIPUMY_API_KEY=
IPQS_API_KEY=
OMKAR_API_KEY=

Leave unused keys blank. Never commit .env or Streamlit secrets to GitHub.

Recommended lookup behavior

  1. User selects one live provider.
  2. The app calls only that provider.
  3. On authentication error, rate limit, timeout, network failure, or server error, use the local processed phone dataset.
  4. If no local record exists, return Unknown, not Safe.
PenipuMY, IPQS, and Carrier Lookup return different kinds of evidence. PenipuMY focuses on scam reports and reputation, IPQS provides validation, fraud-risk, and abuse metadata, while Carrier Lookup provides carrier and line-type context. Always label the selected provider and evidence source clearly.
